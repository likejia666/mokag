> **没有声明的代码 = 无效代码 = 必须重写**

开始写任何代码文件前，必须在回复中声明：

```

- 任务：M1-006
- 锚点：API-6.3.2, DB-users, DB-login_sessions, FLOW-5.2, CODE-10.5
```

## 3. 编码后强制校验（每文件必做）

### 3.1 单文件校验

```
对每个产出的代码文件，逐项检查：

□ 文件头是否有锚点注释？
□ DB锚点：表名/字段名/类型与03-数据库设计一致？
□ API锚点：路径/参数/响应与06-API接口规范一致？
□ FLOW锚点：业务逻辑与05-功能流程图一致？
□ CODE锚点：代码结构与10-开发规范模板一致？
□ SEC锚点：安全处理与08-安全与性能一致？

任一不通过 → 立即修改
```

### 3.2 步骤清单勾选

```
每完成一个步骤，立即在项目进度跟踪文档中勾选对应步骤清单项。
进度值由步骤清单自动推导（已勾选数/总步骤数），禁止手动填写百分比。
```

### 3.3 三级校验（标记✅已完成前必过）

```
L1 基础校验：步骤清单全勾选 + 文件实际存在 + 无语法错误
L2 文档一致性：参数名/类型 + 响应字段/类型 + DB字段/类型 + 业务流转 + 错误码 全部匹配
L3 交叉追溯：关联DB表已创建 + 关联前端/功能模块状态已更新 + 无未记录技术债务

⚠️ L1/L2未通过 → 只能标记 🟡 开发中
⚠️ L3未通过 → 可标记 ✅ 但须登记技术债务
```
