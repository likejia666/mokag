4.1 UI设计规范
--------------

【配色方案】
主色调：
- 主色（Primary）: #5B9EF8 (蓝色，专业可信)
- 辅色（Secondary）: #7C6FE8 (紫色，科技感)
- 成功色（Success）: #52C41A (绿色)
- 警告色（Warning）: #FAAD14 (橙色)
- 错误色（Error）: #F5222D (红色)

背景色：
- 主背景：#F5F7FA (浅灰)
- 卡片背景：#FFFFFF
- 拟态凸起：linear-gradient(145deg, #ffffff, #e6e9ed)
- 拟态凹陷：linear-gradient(145deg, #e6e9ed, #ffffff)

文本色：
- 主文本：#333333
- 次文本：#666666
- 辅助文本：#999999
- 占位文本：#CCCCCC

【拟态风格参数】
圆角：border-radius: 16px (大圆角)
阴影（凸起）：
  box-shadow: 
    6px 6px 12px rgba(0, 0, 0, 0.1),
    -6px -6px 12px rgba(255, 255, 255, 0.8)
阴影（凹陷）：
  box-shadow: 
    inset 4px 4px 8px rgba(0, 0, 0, 0.1),
    inset -4px -4px 8px rgba(255, 255, 255, 0.8)

【字体规范】
标题大：28px/bold
标题中：24px/bold
标题小：20px/bold
正文大：16px/regular
正文中：14px/regular
正文小：12px/regular

【间距规范】
超大间距：32px
大间距：24px
中间距：16px
小间距：12px
超小间距：8px

【布局规范】
- 左上角：抽屉式导航菜单（汉堡图标）
- 顶部：页面标题居中
- 底部：固定导航栏（首页/职位/消息/我的）
- 右下角：页面刷新按钮（圆形悬浮按钮）
- 内容区：卡片式布局，拟态效果

【空状态规范】
所有列表页在无数据时显示空状态：
【UI布局描述：空状态页面】
- 页面中央显示空状态插画区域
- 插画下方显示提示文字"暂无相关内容"
- 提示文字下方显示引导文字"点击右下角刷新试试"
- 整体居中布局，背景为浅色

UI布局（设置新密码页 - 邮件链接进入）：
【UI布局描述：设置新密码页】
- 顶部导航栏：左侧返回按钮，中间标题"设置新密码"
- 新密码输入框（带锁图标，提示6-20位）
- 确认新密码输入框（带锁图标）
- 确认设置按钮

找回密码方式（新增）：
- 客服人工验证：手机号+图形验证码 → 客服核验 → 管理员后台修改密码
- 邮箱自助找回：已绑定邮箱用户可通过邮箱验证码/邮件链接进入设置新密码完成找回

交互逻辑：
1. 登录：
   - 输入手机号、密码、验证码
   - 点击登录→验证→跳转首页
   - 记住登录状态由 system_config.token_expire_days 控制（默认7天）
   - 登录失败达到 system_config.login_fail_max 后→账号锁定 system_config.login_lock_minutes 分钟，显示锁定倒计时
2. 注册：
   - 选择角色类型（求职者/雇主）
   - 填写注册信息
   - 勾选协议→提交注册
   - 注册成功→自动登录→跳转完善信息引导页
3. 找回密码：
   - 顶部提供两种找回方式切换：客服验证 / 邮箱找回
   - 方式A（客服）：
     - 步骤1：输入手机号+图形验证码验证身份
     - 步骤2：联系客服人工验证
     - 客服验证通过后，管理员在后台直接修改用户密码
     - 修改后用户使用新密码登录；如命中安全策略，可要求首次登录后强制修改密码
   - 方式B（邮箱）：
     - 仅限已绑定邮箱用户
     - 步骤1：输入绑定邮箱+图形验证码，提交后调用发送找回邮件接口
     - 步骤2：页面提示邮件已发送，用户查收邮件并点击设置新密码链接
     - 用户通过邮件链接进入【设置新密码页】，输入新密码并确认
     - 重置成功后强制重新登录；旧 token / 会话自动失效

颜色设计：
- 背景：#F5F7FA
- 卡片：拟态效果白色
- 按钮：渐变蓝色 #5B9EF8
- 输入框：拟态凹陷效果
- 锁定提示：红色背景警告框
