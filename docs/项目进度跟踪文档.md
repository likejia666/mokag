## 3. 步骤模板定义

> **📎 本章节定义3种API步骤模板。AI编码时必须按对应模板的步骤顺序执行，不得跳步。**
> **百分比由步骤清单自动推导（已勾选数/总步骤数），禁止手动填写。**

### 3.1 模板分类规则

| API类型 | 判断依据 | 步骤数 |
|---------|---------|--------|
| **CRUD型** | 含写操作（POST/PUT/DELETE）的接口 | 9步 |
| **查询型** | 只读接口（GET列表/详情） | 6步 |
| **操作型** | 状态变更/动作类接口（登录/审核/置顶/举报等） | 7步 |

### 3.2 CRUD型步骤模板（9步）

```
☐ Step 1: Migration — 表创建/字段追加（锚点DB-xxx）

☐ Step 3: Request — FormRequest字段校验规则（锚点API-xxx）
☐ Step 4: Service — 业务逻辑+事务边界+状态机校验+事件派发（锚点API-xxx, FLOW-xxx）
☐ Step 5: Controller — 调度层（调用Service+返回Resource）（锚点API-xxx, CODE-xxx）
☐ Step 6: Resource — API Resource响应格式化+脱敏+File Resource嵌套（锚点API-xxx）
☐ Step 7: Policy — 资源级权限判断（锚点API-xxx, SEC-xxx）
☐ Step 8: Route — 路由注册+中间件+限流配置
☐ Step 9: 自检 — 对照API-xxx校验(1)请求参数名/类型一致 (2)响应字段名/类型一致 (3)错误码一致
```
