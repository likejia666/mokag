9.1 服务器要求
--------------------------------------------------

【硬件配置】
- CPU：4核及以上
- 内存：8GB及以上
- 硬盘：100GB及以上SSD

【软件环境 - Laravel 11.x 要求】
- 操作系统：Linux (Ubuntu 22.04+/CentOS 8+/Debian 11+)
- Web服务器：Nginx 1.20+ 或 Apache 2.4+
- PHP：8.2+ (Laravel 11.x 最低要求)
- MySQL：8.0+ 或 MariaDB 10.6+
- Redis：6.0+
- Composer：2.5+

【PHP 8.2 必需扩展】
- php8.2-fpm
- php8.2-mysql (pdo_mysql)
- php8.2-redis
- php8.2-mbstring
- php8.2-json
- php8.2-openssl
- php8.2-ctype
- php8.2-xml
- php8.2-bcmath
- php8.2-tokenizer
- php8.2-fileinfo
- php8.2-curl

【Laravel 11.x 目录结构要求】
- 应用代码：/path/to/project/
- Web根目录：/path/to/project/public/ (Nginx root指向此处)

- 缓存目录：/path/to/project/bootstrap/cache/ (需可写权限)

9.2 部署步骤
--------------------------------------------------

【1. 环境准备 - PHP 8.2】

1. 安装PHP 8.2及扩展 (Ubuntu示例)
   sudo apt update
   sudo apt install php8.2-fpm php8.2-mysql php8.2-redis php8.2-mbstring \
                    php8.2-json php8.2-openssl php8.2-ctype php8.2-xml \
                    php8.2-bcmath php8.2-tokenizer php8.2-fileinfo php8.2-curl

2. 安装Composer
   curl -sS https://getcomposer.org/installer | php
   sudo mv composer.phar /usr/local/bin/composer

3. 安装MySQL 8.0+
   sudo apt install mysql-server-8.0
   - 创建数据库：CREATE DATABASE m_mokag_top CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
   - 创建用户并授权

4. 安装Redis 6.0+
   sudo apt install redis-server
   - 配置持久化：appendonly yes
   - 设置密码：requirepass your_password

5. 安装Nginx 1.20+
   sudo apt install nginx
   - 配置虚拟主机
   - 配置SSL证书

【2. Laravel 项目部署】

1. 上传代码到服务器
   cd /var/www
   git clone https://your-repo.git m_mokag_top
   cd m_mokag_top

2. 安装PHP依赖
   composer install --no-dev --optimize-autoloader

3. 配置环境文件
   cp .env.example .env
   php artisan key:generate
   - 编辑 .env 文件配置数据库、Redis等

4. 配置数据库连接 (.env)
   DB_CONNECTION=mysql
   DB_HOST=127.0.0.1
   DB_PORT=3306
   DB_DATABASE=m_mokag_top
   DB_USERNAME=your_username
   DB_PASSWORD=your_password

5. 配置Redis连接 (.env)
   REDIS_HOST=127.0.0.1
   REDIS_PASSWORD=your_password
   REDIS_PORT=6379

6. 执行数据库迁移
   php artisan migrate

7. 执行数据填充（如有）
   php artisan db:seed

8. 创建存储链接
   php artisan storage:link

9. 设置文件权限
   chmod -R 755 /var/www/m_mokag_top
   chmod -R 775 /var/www/m_mokag_top/storage
   chmod -R 775 /var/www/m_mokag_top/bootstrap/cache
   chown -R www-data:www-data /var/www/m_mokag_top

10. 优化Laravel性能
    php artisan config:cache
    php artisan route:cache
    php artisan view:cache

【3. Nginx配置 - Laravel】
server {
    listen 80;
    listen 443 ssl http2;
    server_name m.mokag.top;
    
    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;
    
    # Laravel根目录指向public
    root /var/www/m_mokag_top/public;
    index index.php index.html;
    
    # Laravel重写规则
    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }
    
    # PHP 8.2-FPM配置
    location ~ \.php$ {
        fastcgi_pass unix:/run/php/php8.2-fpm.sock;
        fastcgi_index index.php;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
        include fastcgi_params;
        
        # Laravel优化
        fastcgi_buffer_size 32k;
        fastcgi_buffers 8 16k;
    }
    
    # 禁止访问敏感文件
    location ~ /\.(?!well-known).* {
        deny all;
    }
    
    # WebSocket代理
    location /ws {
        proxy_pass http://127.0.0.1:7272;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_read_timeout 86400;
    }
}

【4. WebSocket服务部署】
1. 安装 Workerman 相关依赖（Laravel 项目内通过 Composer 管理）
   composer require workerman/workerman
   composer require workerman/gateway-worker

2. 约定独立启动脚本
   - WebSocket 常驻进程建议放在 `websocket/start.php` 或 `bin/websocket.php`
   - 该脚本负责初始化 Laravel 容器/环境变量，并启动 Gateway、BusinessWorker、Register 等进程

3. 启动 Workerman 服务
   php websocket/start.php start -d
   # 停止
   php websocket/start.php stop
   # 重启
   php websocket/start.php restart -d

4. 配置开机自启 (Systemd)
   sudo nano /etc/systemd/system/workerman.service
   
   [Unit]
   Description=Workerman WebSocket Server
   After=network.target
   
   [Service]
   Type=forking
   User=www-data
   WorkingDirectory=/var/www/m_mokag_top
   ExecStart=/usr/bin/php /var/www/m_mokag_top/websocket/start.php start -d
   ExecStop=/usr/bin/php /var/www/m_mokag_top/websocket/start.php stop
   ExecReload=/usr/bin/php /var/www/m_mokag_top/websocket/start.php restart -d
   Restart=always
   
   [Install]
   WantedBy=multi-user.target
   
   sudo systemctl daemon-reload
   sudo systemctl enable workerman
   sudo systemctl start workerman

5. 使用 Supervisor 管理进程（备选方案）
   [program:workerman]
   command=php /var/www/m_mokag_top/websocket/start.php start
   directory=/var/www/m_mokag_top
   autostart=true
   autorestart=true
   user=www-data
   redirect_stderr=true
   stdout_logfile=/var/log/workerman.log

说明：
- 文档默认采用独立启动脚本方式管理 Workerman，不假定项目内已额外封装 `php artisan workerman:start` 命令
- Nginx 对外统一暴露 `wss://m.mokag.top/ws`，内部代理到 Workerman 实际监听地址（如 127.0.0.1:7272）

【5. Laravel定时任务】
配置 crontab（Laravel 使用 Artisan 调度器）：
# 编辑 crontab
sudo crontab -e

# 添加 Laravel 调度器（每分钟执行）
* * * * * cd /var/www/m_mokag_top && php artisan schedule:run >> /dev/null 2>&1

# Laravel 11 建议在 routes/console.php 中定义调度任务，例如：
use Illuminate\Support\Facades\Schedule;

Schedule::command('app:clean-expired-data')->dailyAt('03:00');
Schedule::command('app:clean-expired-top')->hourly();
Schedule::command('app:clean-logs')->weeklyOn(0, '01:00');

说明：
- 自定义命令通过 `php artisan make:command` 创建，命令类位于 app/Console/Commands/
- 调度定义可集中维护在 routes/console.php，也可在服务提供者中按模块拆分注册
- 部署后如使用 `schedule:work` 常驻模式，应与 Supervisor/Systemd 配套管理

【6. Laravel队列配置】
1. 配置队列驱动 (.env)
   QUEUE_CONNECTION=redis
   
2. 启动队列工作进程（Supervisor）
   [program:laravel-worker]
   process_name=%(program_name)s_%(process_num)02d
   command=php /var/www/m_mokag_top/artisan queue:work redis --sleep=3 --tries=3 --max-time=3600 --queue=default,notifications
   autostart=true
   autorestart=true
   stopasgroup=true
   killasgroup=true
   user=www-data
   numprocs=2
   redirect_stderr=true
   stdout_logfile=/var/log/laravel-worker.log
   stopwaitsecs=3600

说明：
- 队列任务建议按业务拆分队列名称，如 notifications、audit、export、chat
- 发布新代码后执行 `php artisan queue:restart`，让 Worker 平滑重载新版本代码
- 涉及超时、重试、限流、中间件等控制时，优先在 Job 类中显式配置

【7. 部署后验证】

2. 检查Laravel状态

3. 检查数据库连接

9.3 数据备份
--------------------------------------------------

【数据库备份】

安全要求：
- 禁止在命令行、crontab、脚本中明文写入数据库密码（防止密码暴露于进程列表、命令历史、运维平台记录）
- 使用专用备份账号（最小权限：SELECT、LOCK TABLES、SHOW VIEW、EVENT、TRIGGER）
- 凭据存储于受限权限配置文件（chmod 600），不随代码提交

推荐方式：使用 ~/.my.cnf 凭据文件

# /etc/mysql/backup.cnf 或 /root/.my.cnf（chmod 600，仅 root 可读）
[mysqldump]
user=backup_user
password=your_backup_password
host=127.0.0.1

# 每天凌晨2点自动备份（crontab）
0 2 * * * mysqldump --defaults-extra-file=/etc/mysql/backup.cnf m_mokag_top | gzip > /backup/db_$(date +\%Y\%m\%d).sql.gz

备份安全增强：
- 备份文件加密（可选）：openssl enc -aes-256-cbc -in backup.sql.gz -out backup.sql.gz.enc
- 备份完整性校验：生成并保存 SHA256 校验文件
  sha256sum /backup/db_$(date +\%Y\%m\%d).sql.gz > /backup/db_$(date +\%Y\%m\%d).sql.gz.sha256
- 备份文件权限：chmod 640 /backup/db_*.sql.gz
- 异地/对象存储同步：备份完成后同步至私有对象存储桶（如 rclone / aws s3 cp）
- 恢复演练：每季度执行一次恢复测试，验证备份可用性

【文件备份】
# 每周日凌晨1点备份 Laravel 公共存储目录
0 1 * * 0 tar -czf /backup/storage_public_$(date +\%Y\%m\%d).tar.gz /path/to/project/storage/app/public

【备份保留】
- 数据库备份保留30天
- 文件备份保留90天
- 异地备份策略
