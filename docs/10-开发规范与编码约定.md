10.2 命名约定
------------------

【统一命名规范】

| 对象 | 命名风格 | 示例 | 禁止 |
|------|----------|------|------|
| 数据库表名 | snake_case 复数 | users, job_categories | User, jobCategorys |
| 数据库字段名 | snake_case | created_at, is_deleted | createdAt, IsDeleted |
| 模型类名 | PascalCase 单数 | User, JobCategory | user, JobCategories |
| 控制器类名 | PascalCase + Controller | AuthController | auth_controller |
| Service类名 | PascalCase + Service | AuthService | auth_service |
| FormRequest类名 | PascalCase + Request | RegisterRequest | register_request |
| Resource类名 | PascalCase + Resource | UserResource | user_resource |
| Policy类名 | PascalCase + Policy | JobPolicy | job_policy |
| PHP方法名 | camelCase | getUserInfo | get_user_info, GetUserInfo |
| PHP变量名 | camelCase | $userName | $user_name, $UserName |
| PHP常量名 | UPPER_SNAKE_CASE | MAX_LOGIN_ATTEMPTS | maxLoginAttempts |
| Enum类名 | PascalCase | JobStatus | job_status |
| 路由路径 | kebab-case | /api/job/list | /api/jobList, /api/job_list |
| 路由参数 | kebab-case | {job-id} | {jobId}, {job_id} |
| 配置键名 | snake_case 点分 | auth.jwt.ttl | authJwtTtl |
| 事件名（WS） | 点分小写 | message.created | messageCreated, MESSAGE_CREATED |
| 迁移文件 | 时间戳_操作_表名 | 2026_04_16_000001_create_users_table | createUsers |
| Seeder类名 | PascalCase + Seeder | SystemConfigSeeder | system_config_seeder |
| Job类名 | PascalCase + Job | ProcessExport | process_export |

【数据库命名特别约定】
- 主键统一使用 id（Laravel $table->id()，bigint unsigned 自增）
- 外键字段使用 `{关联表名单数}_id`：如 user_id, job_id, category_id
- 布尔字段使用 is_ 前缀：is_deleted, is_top, is_enabled
- 时间字段使用 _at 后缀：created_at, updated_at, deleted_at, expire_at
- 计数缓存字段使用 _count 后缀：views_count, comments_count
- JSON字段不得使用 text 类型替代，必须显式使用 JSON 类型
- 状态字段如为 ENUM，值统一使用小写蛇形：pending, active, rejected

10.3 代码分层规范
------------------

【强制分层架构】

```
请求流入方向：

HTTP Request
    │
    ▼
Route（路由定义 + 中间件分配）
    │
    ▼
FormRequest（参数校验 + 授权预检）
    │
    ▼
Controller（调度层：接收请求、调用Service、返回Resource）
    │
    ▼
Service（业务逻辑层：核心业务处理、状态流转、事务管理）
    │
    ▼
Model（数据访问层：Eloquent ORM、查询作用域、访问器/修改器）
    │
    ▼
Database
```

【各层职责边界 — 严禁越界】

| 层 | 允许做什么 | 禁止做什么 |
|----|-----------|-----------|
| **Controller** | 接收请求、调用FormRequest校验、调用Service、组装Resource响应、事务边界声明 | ❌ 写业务逻辑、❌ 直接操作Model查询、❌ 直接返回数组 |
| **Service** | 业务逻辑、状态流转、数据校验（业务级）、事务管理、调用其他Service、触发事件 | ❌ 依赖Request对象、❌ 返回HTTP响应、❌ 直接读取用户输入 |
| **Model** | 定义表结构、关联关系、查询作用域、访问器/修改器、$casts、$fillable | ❌ 写业务逻辑、❌ 调用Service、❌ 处理HTTP请求 |
| **FormRequest** | 参数校验规则、字段白名单、授权预检、自定义错误消息 | ❌ 写业务逻辑、❌ 修改数据 |
| **Resource** | 响应字段映射、脱敏、枚举值转换、嵌套资源 | ❌ 写业务逻辑、❌ 执行查询 |
| **Policy** | 资源级权限判断（能否编辑、能否审核等） | ❌ 写业务逻辑、❌ 修改数据 |

10.4 Controller 规范
------------------

【Controller 编写规范】

1. Controller 仅做调度，方法体不超过 15 行
2. 每个方法只做一件事：接收 → 调用Service → 返回Resource
3. 使用依赖注入获取Service实例
4. 使用Route Model Binding
5. 禁止在Controller中写DB查询

```php
<?php

declare(strict_types=1);

namespace App\Http\Controllers\Api\User;

use App\Http\Controllers\Controller;
use App\Http\Requests\Api\User\RegisterRequest;
use App\Http\Resources\Api\UserResource;
use App\Services\AuthService;
use Illuminate\Http\JsonResponse;

class AuthController extends Controller
{
    public function __construct(
        private readonly AuthService $authService,
    ) {}

    /**
     * 用户注册 (6.1.2)
     *
     * 对应文档：06-API接口规范-上 → 6.1.2 用户注册
     */
    public function register(RegisterRequest $request): JsonResponse
    {
        $user = $this->authService->register($request->validated());

        return (new UserResource($user))
            ->response()
            ->setStatusCode(201);
    }

    /**
     * 用户登录 (6.1.3)
     *
     * 对应文档：06-API接口规范-上 → 6.1.3 用户登录
     */
    public function login(LoginRequest $request): JsonResponse
    {
        $result = $this->authService->login($request->validated());

        return response()->json([
            'code' => 200,
            'msg' => 'success',
            'data' => $result,
        ]);
    }
}
```

10.5 Service 规范
------------------

【Service 编写规范】

1. Service 不依赖 Request / Response 对象
2. Service 方法接收纯数据（数组或DTO），返回 Model 或纯数据
3. 多表写操作必须使用 DB 事务
4. 状态流转必须有明确的状态机校验
5. Service 间调用通过依赖注入，禁止 new 其他Service

```php
<?php

declare(strict_types=1);

namespace App\Services;

use App\Enums\UserRole;
use App\Enums\UserStatus;
use App\Models\User;
use Illuminate\Support\Facades\DB;
use Illuminate\Support\Facades\Hash;
use Illuminate\Validation\ValidationException;

class AuthService
{
    public function __construct(
        private readonly UserService $userService,
    ) {}

    /**
     * 用户注册
     *
     * @param array<string, mixed> $data 已校验的注册数据
     * @return User
     * @throws ValidationException
     */
    public function register(array $data): User
    {
        return DB::transaction(function () use ($data) {
            // 创建用户
            $user = User::create([
                'phone' => $data['phone'],
                'password' => Hash::make($data['password']),
                'role' => $data['role'],
                'nickname' => $data['nickname'] ?? '用户' . substr($data['phone'], -4),
                'ip_address' => $data['ip_address'] ?? null,
                'device_id' => $data['device_id'] ?? null,
            ]);

            // 雇主角色自动创建企业记录
            if ($user->role === UserRole::EMPLOYER->value) {
                $user->company()->create([
                    'audit_status' => 0, // 未提交
                ]);
            }

            return $user;
        });
    }

    /**
     * 用户登录
     *
     * @param array<string, mixed> $data 已校验的登录数据
     * @return array<string, mixed> Token和用户信息
     * @throws ValidationException
     */
    public function login(array $data): array
    {
        $user = User::where('phone', $data['phone'])->first();

        // 状态校验
        if (!$user || !Hash::check($data['password'], $user->password)) {
            throw ValidationException::withMessages([
                'phone' => ['手机号或密码错误'],
            ]);
        }

        // 账号状态检查
        $this->checkAccountStatus($user);

        // 生成Token
        $token = $user->createToken('auth_token')->plainTextToken;

        return [
            'token' => $token,
            'token_type' => 'Bearer',
            'expires_in' => config('auth.jwt.ttl', 7200),
            'user' => new \App\Http\Resources\Api\UserResource($user),
        ];
    }

    /**
     * 检查账号状态
     *
     * @throws ValidationException
     */
    private function checkAccountStatus(User $user): void
    {
        // 优先级：已注销 > 锁定 > 黑名单 > 注销申请中
        if ($user->is_deleted === 2) {
            throw ValidationException::withMessages([
                'phone' => ['该账号已注销'],
            ]);
        }

        if ($user->login_locked_until && $user->login_locked_until->isFuture()) {
            throw ValidationException::withMessages([
                'phone' => ['账号已被锁定，请稍后再试'],
            ]);
        }

        if ($user->is_blacklist === 1) {
            throw ValidationException::withMessages([
                'phone' => ['该账号已被封禁'],
            ]);
        }

        if ($user->is_deleted === 1) {
            throw ValidationException::withMessages([
                'phone' => ['该账号正在注销中'],
            ]);
        }
    }
}
```

10.6 Model 规范
------------------

【Model 编写规范】

1. 必须声明 $table 属性（即使与默认一致，显式声明防止歧义）
2. 必须声明 $fillable 或 $guarded
3. 必须声明 $casts 进行类型转换
4. 枚举字段使用 PHP 8.2 Enum
5. 软删除使用 SoftDeletes trait
6. 关联关系必须定义
7. 查询作用域按需定义

```php
<?php

declare(strict_types=1);

namespace App\Models;

use App\Enums\JobStatus;
use Illuminate\Database\Eloquent\Builder;
use Illuminate\Database\Eloquent\Model;
use Illuminate\Database\Eloquent\SoftDeletes;

class Job extends Model
{
    use SoftDeletes;

    protected $table = 'jobs';

    protected $fillable = [
        'job_no',
        'user_id',
        'company_id',
        'title',
        'work_location',
        'work_period',
        'work_time',
        'settlement_type',
        'accommodation',
        'meals',
        'requirements',
        'education',
        'age_range',
        'tags',
        'images',
        'contact_person',
        'contact_phone',
        'status',
        'close_reason',
        'is_top',
        'top_expire_time',
        'audit_status',
        'audit_reason',
        'expire_time',
        'refreshed_at',
    ];

    protected $casts = [
        'images' => 'array',
        'is_top' => 'boolean',
        'accommodation' => 'boolean',
        'meals' => 'boolean',
        'views_count' => 'integer',
        'comments_count' => 'integer',
        'top_expire_time' => 'datetime',
        'expire_time' => 'datetime',
        'refreshed_at' => 'datetime',
    ];

    /**
     * 状态枚举映射（03-数据库设计 → 核心业务状态机总则）
     */
    public const STATUS_DRAFT = 'draft';
    public const STATUS_PENDING = 'pending';
    public const STATUS_REJECTED = 'rejected';
    public const STATUS_ACTIVE = 'active';
    public const STATUS_PAUSED = 'paused';
    public const STATUS_EXPIRED = 'expired';
    public const STATUS_CLOSED = 'closed';
    public const STATUS_DELETED = 'deleted';

    /**
     * 关闭原因枚举
     */
    public const CLOSE_REASON_MANUAL = 'manual';
    public const CLOSE_REASON_FILLED = 'filled';
    public const CLOSE_REASON_ADMIN = 'admin';

    // ─── 关联关系 ───

    public function user(): BelongsTo
    {
        return $this->belongsTo(User::class);
    }

    public function company(): BelongsTo
    {
        return $this->belongsTo(Company::class);
    }

    public function jobLevels(): HasMany
    {
        return $this->hasMany(JobLevel::class);
    }

    public function applications(): HasMany
    {
        return $this->hasMany(Application::class);
    }

    // ─── 查询作用域 ───

    public function scopeActive(Builder $query): Builder
    {
        return $query->where('status', self::STATUS_ACTIVE);
    }

    public function scopeNotDeleted(Builder $query): Builder
    {
        return $query->where('status', '!=', self::STATUS_DELETED);
    }

    public function scopeTopFirst(Builder $query): Builder
    {
        return $query->orderByDesc('is_top')->orderByDesc('refreshed_at');
    }
}
```

10.7 FormRequest 规范
------------------

【FormRequest 编写规范】

1. 所有接口必须使用 FormRequest 校验
2. rules() 返回校验规则
3. authorize() 返回授权判断
4. messages() 返回自定义错误消息
5. 禁止在 Controller 中使用 $request->validate()

```php
<?php

declare(strict_types=1);

namespace App\Http\Requests\Api\User;

use Illuminate\Foundation\Http\FormRequest;

class RegisterRequest extends FormRequest
{
    /**
     * 授权检查
     */
    public function authorize(): bool
    {
        return true; // 注册接口无需授权
    }

    /**
     * 校验规则
     *
     * 对应文档：06-API接口规范-上 → 6.1.2 用户注册
     */
    public function rules(): array
    {
        return [
            'phone' => ['required', 'regex:/^1[3-9]\d{9}$/', 'unique:users,phone'],
            'password' => ['required', 'string', 'min:6', 'max:20', 'confirmed'],
            'password_confirmation' => ['required', 'string'],
            'role' => ['required', 'in:seeker,employer'],
            'code' => ['required', 'string', 'size:6'],
            'nickname' => ['nullable', 'string', 'max:50'],
            'device_id' => ['nullable', 'string', 'max:100'],
        ];
    }

    /**
     * 自定义错误消息
     */
    public function messages(): array
    {
        return [
            'phone.required' => '手机号不能为空',
            'phone.regex' => '手机号格式不正确',
            'phone.unique' => '该手机号已注册',
```

10.8 API Resource 规范
------------------

【Resource 编写规范】

1. 所有接口响应必须通过 Resource 组装
2. 禁止直接返回 Model 或数组
3. 脱敏逻辑放在 Resource 中
4. 枚举值转换放在 Resource 中
5. 嵌套资源使用 ResourceCollection

```php
<?php

declare(strict_types=1);

namespace App\Http\Resources\Api;

use Illuminate\Http\Request;
use Illuminate\Http\Resources\Json\JsonResource;

class UserResource extends JsonResource
{
    /**
     * 用户信息资源
     *
     * 对应文档：06-API接口规范-上 → 6.2.1 获取用户信息
     */
    public function toArray(Request $request): array
    {
        return [
            'id' => $this->id,
            'phone' => $this->phone_masked,    // 脱敏：138****0000
            'role' => $this->role,
            'nickname' => $this->nickname,
            'avatar' => $this->when(
                $this->avatar_file_id,
                fn () => new FileResource($this->whenLoaded('avatarFile'))
            ),
            'email' => $this->email,
            'is_certified' => $this->is_certified,
            'certified_status' => match((int) $this->is_certified) {
                0 => 'not_submitted',
                1 => 'pending',
                2 => 'approved',
                3 => 'rejected',
                default => 'unknown',
            },
            'must_change_password' => (bool) $this->must_change_password,
            'created_at' => $this->created_at?->format('Y-m-d H:i:s'),
        ];
    }
}
```

10.9 Policy 规范
------------------

```php
<?php

declare(strict_types=1);

namespace App\Policies;

use App\Models\Job;
use App\Models\User;

class JobPolicy
{
    /**
     * 是否可编辑职位
     */
    public function update(User $user, Job $job): bool
    {
        return $user->id === $job->user_id;
    }

    /**
     * 是否可删除职位
     */
    public function delete(User $user, Job $job): bool
    {
        return $user->id === $job->user_id;
    }

    /**
     * 是否可刷新职位
     */
    public function refresh(User $user, Job $job): bool
    {
        return $user->id === $job->user_id
            && $job->status === Job::STATUS_ACTIVE;
    }
}
```

10.10 事务边界规范
------------------

【事务使用规则】

1. **涉及多表写操作必须使用事务**
2. **事务在 Service 层管理，Controller 不管理事务**
3. **事务内禁止调用外部HTTP接口**
4. **事务内禁止发送异步通知（使用事件监听在事务后执行）**

必须使用事务的场景：
- 用户注册（users + companies）
- 职位发布（jobs + job_levels + job_images）
- 报名创建（applications + job_levels.applied_count更新）
- 消息发送（messages_private + conversations更新）
- 企业认证提交（companies + company_audit_histories）
- 审核操作（目标表 + operation_logs + notifications）
- 报名审核（applications状态 + notifications + job_levels计数更新）

```php
// 正确：事务在Service中
public function createJob(array $data): Job
{
    return DB::transaction(function () use ($data) {
        $job = Job::create([...]);
        foreach ($data['levels'] as $levelData) {
            $job->jobLevels()->create([...]);
        }
        return $job;
    });
}
```
