## 重要开发约束备忘

> 以下为技术文档中明确要求的关键约束，开发时必须遵守

1. **框架规范**：所有数据库变更通过 Migration 管理，禁止直接手改生产表；路由按 api.php/console.php 分层；中间件在 bootstrap/app.php 注册。
2. **响应格式**：所有 API 统一返回 `{code, msg, data}`，错误码严格按 §6.2 错误码规范。
3. **JWT 双Guard**：用户端 guard:api，管理员端 guard:admin，Token 黑名单存 Redis。
4. **内容安全**：评论/私聊/职位描述发布前必须经过 ContentSafetyService 统一检测。
5. **报名事务**：提交报名/取消报名/审核报名均需数据库事务，applied\_count 原子更新。
6. **浏览防刷**：job\_view\_dedupe Redis key，TTL 5分钟，viewer\_key = user\_id 或 IP。
7. **消息幂等**：client\_msg\_id 唯一索引，防止重复发送。
8. **草稿清理**：draft\_retention\_days 从 system\_config 读取，默认7天。
9. **消息过期**：expire\_at = created\_at + message\_retention\_days(默认30天)，由消息服务统一写入。
10. **脱敏规则**：手机号未授权场景138\*\*\*\*8888；id\_card AES-256加密存储；导出文件敏感数据脱敏。
11. **复杂统计**：聚合查询由独立 Service（JobStatisticsService/AdminStatisticsService）处理，不放在 Controller。
12. **部署规范**：部署后统一执行 migrate、config:cache、queue:restart、storage:link、健康检查。
13. **联系方式过滤**：system\_config 中 `contact_filter_enabled=1` 时开启；过滤规则由 `contact_filter_pattern` 配置；默认过滤手机号（1\[3-9]\d{9}）、微信/QQ词汇；触发时返回错误码 7004；由 ContactFilterService 统一处理。
14. **WebSocket鉴权**：连接时通过 URL 参数携带 `ws_token`；鉴权失败服务端主动关闭连接并返回错误码 9001；由 Workerman + GatewayWorker 承担实时连接，Nginx 对外统一暴露 `/ws`；心跳间隔30秒；断线重连前必须同时校验 `ws_token` 与 `device_session_id` 是否仍有效。
15. **文件模型**：业务层统一传 `file_id`，Service 层校验 files 表归属与类型；对外统一返回 File Resource，私有文件通过 private disk + temporary\_url 输出，不直接暴露存储路径。
16. **审计日志**：后台所有高危写操作必须写入 `operation_logs`，成功/失败/拒绝三类结果均需留痕；配置变更除 `config_change_logs` 外还必须同步写入 `operation_logs`。
